# 🍉 AIR FRUIT NINJA — PROMPT + WORKING

## ✅ Prompt for Replit AI

Create a **gesture-controlled Fruit Ninja style game** using webcam input and hand tracking.

The interface must show:

* Live camera feed inside a rounded container
* A **motion trail following the hand (blade effect)**
* Floating fruits with smooth animations
* Real-time score, lives, and combo display

---

### 🎮 Core Interaction

* The player uses their **hand as a blade**
* Fast hand movement = slicing action
* Slow movement = ignored

---

### ⚙️ System Behavior

#### Hand Tracking

* Track hand continuously
* Show a glowing trail behind movement

---

#### Swipe Detection

* Detect fast movement between frames
* Activate slicing only when speed crosses threshold

---

#### Fruit Behavior

* Fruits spawn from bottom
* Move upward and fall naturally
* Multiple fruits at once

---

#### Collision System

* Detect if **hand movement path intersects fruit**
* Not just point touch — use motion path

---

#### Slice Result

* Fruit splits into 2 parts
* Pieces fall down
* Juice splash effect
* Sound plays
* Score increases

---

#### Bomb Behavior

* Random bombs appear
* If sliced:

  * Explosion animation
  * Game over or heavy penalty

---

### 🎯 UI Requirements

* Top bar:

  * Score
  * Lives (hearts)
  * Combo indicator

* Visual feedback:

  * Motion trail
  * Slice effect
  * Explosion effect

---

### 🚀 Quality Requirements

* Smooth animations
* No lag
* Accurate slicing
* Real-time responsiveness

---

## 🎮 How It Should Work (Clear Understanding)

* Hand moves → system tracks path
* If movement is fast → slicing mode ON
* That movement line cuts through fruits
* Each intersected fruit → sliced instantly
* Missing fruits → life decreases
* Bomb hit → game ends

👉 The key is **motion-based interaction, not touch**
